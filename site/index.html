Philosophy

A foundation worth building on.

Available everywhere, always, and instantly — with data that stays on the device. No account required. No server in the way.

Offline first

Works after the first load with no network. Service Worker, cache-first strategy, and update flow are included and configured from day one.

Mobile first

Built for touch from day one: safe areas, gesture library, deferred SW updates that never interrupt a session. Installable as a PWA to the home screen.

User data stays theirs

Data lives in IndexedDB on the user's device. No server, no accounts, ever. Users export or import data as needed — the library ships the tools to do it.

Built to last

Vanilla HTML, CSS, and JS. No framework version churn, no tool configuration drift. What you write today works in ten years.

No runtime dependencies

Code is copied into your project. You own it, read it, modify it. No upstream to break you, no supply chain risk, no abstraction you can't see through.

Copy, don't import

_lib/ is yours. npx socle update replaces it; your app/ code is never touched. Review every change, update on your terms.

Only what you need

Optional modules — gestures, sync, images, dialogs — are selected at scaffold time and copied in. No dead code shipped, no tree-shaking required.

Features

Build the app. Not the infrastructure.

Everything underneath is tested and ready to extend — so you can focus on what makes your app unique.

PWA

Installable to home screen, manifest, Service Worker, offline caching — all included.

Web Components

AppElement base class, Shadow DOM, three tiers: Page, UI, and Service.

Gesture library

Tap, long press, swipe, hold-drag. Mixin for host elements, Gestures.attach for sub-elements.

Deferred updates

User taps to apply SW updates. Session never interrupted automatically.

Router

History API, SW navigation intercept, animated page transitions, no hash URLs.

Two store models

Event log — full history, P2P-ready. Simple key-value state — coming soon.

Data migrations

Schema versioning and migration runner at boot, before any UI renders.

Multilingual

Locale registry, t() helper, locale switching with localStorage persistence.

CSS token system

Retheme any app by changing two custom properties. RTL-ready logical properties throughout.

Build & deploy

Content-hashed assets, version.json, GitHub Actions deploy workflow — included.

Claude Code skills

Slash commands to scope, build, test, review, and contribute back to the library.

P2P data exchange

WebRTC / WS local network sync, hub mode for competition apps.

Get started

One command does everything

The CLI installs the library, prompts for your setup, and generates a fully working app — no separate install step required.

Step 01

Scaffold & install

Run the CLI directly from GitHub — no global install required. It prompts for your app name, optional modules, and accent colour, then creates ./my-app/ with everything wired up. 

npx github:magp/Socle my-app

Step 02

Build & run locally

Your code lives in app/. The library lives in _lib/. The build script hashes assets, generates the Service Worker, and writes dist/.

cd my-app
npm install
node utils/build.js
npx serve dist --single

Step 03

Deploy to GitHub Pages

Push to main. The included GitHub Actions workflow builds and deploys dist/ automatically. Enable Pages in your repo settings once — every push ships thereafter.

# /i18n

Audit a file for hardcoded user-visible strings.

## Usage
/i18n <filepath>

---

## What counts as a user-visible string

Strings that must not be hardcoded if you plan to support multiple languages:
- Button labels, link text, headings, status messages
- Placeholder text, error messages, confirmation prompts
- Any text a user reads or acts on

Strings that are fine to hardcode:
- HTML tag names, attribute names, CSS class names
- IDB object store names, store action types, event names
- Developer-facing console errors

---

## Audit

### 1. Scan for hardcoded strings

Read every `template()` method and any string used in a DOM update. Flag any string literal that is user-visible text.

If the app is single-language and has no plans for i18n, hardcoded strings in `app/` are acceptable. Flag them as warnings, not failures.

If `app/strings.js` exists, every user-visible string must go through `t()`:

```js
// ❌ Hardcoded — fails if i18n is in use
template() {
  return `<button>Reload</button>`;
}

// ✅ Correct — passes
template() {
  return `<button>${t('reload')}</button>`;
}
```

### 2. Check `app/strings.js` completeness

If `app/strings.js` exists, verify:
- Every key used in a `t()` call across `app/` has a registered value
- `app/strings.js` is the first import in `app/main.js` — strings must be registered before components render
- No key is registered but never used (dead string)

### 3. Check locale key parity (if multiple locales exist)

If `app/locales/` exists with more than one locale file:
- Every key in every locale file must exist in `en.js`
- Every key in `en.js` must exist in every other locale file

Report:
```
Missing in fr.js: reload, dismiss
Extra in fr.js (not in en.js): fermer   ← likely a renamed key
```

---

## Report format

- ✅ Pass — no hardcoded strings (or single-language app, strings noted as acceptable)
- ⚠️ Warning — hardcoded string in a single-language app (acceptable but noted)
- ❌ Fail — hardcoded string when `app/strings.js` is in use, or missing key registration

End with: how many failures (blocking), how many warnings.
